# Check Strategies

開発時に参照する戦略ファイル。新しいパターンを発見したら更新する。

## 開発時の参照

コードを変更・追加する際、以下の手順で戦略を確認する。

### Step 1: ファイル一覧を確認

```bash
ls .claude/strategies/
```

### Step 2: 必要な戦略ファイルを読む

作業内容に関連するファイルを読み、テンプレートや命名規則に従う。

例:
- ルート作成 → `route.md`
- コンポーネント作成 → `component.md`
- 全体把握 → `overview.md`

## 戦略の更新

開発中に以下を発見したら、該当する戦略ファイルを更新する。

- 新しい実装パターン
- 既存パターンの改善
- 新しい命名規則
- 新しい禁止事項

### 更新の判断基準

- 2回以上使われるパターン → 追加を検討
- 既存パターンと矛盾 → 既存を更新
- プロジェクト固有の制約 → 注意事項に追加

### 更新フォーマット

戦略ファイルは以下の構造を維持する。

```markdown
# カテゴリ名

概要（1-2文）

## 配置場所

ディレクトリ構造

## 命名規則

| パターン | 例 |
|---------|-----|

## テンプレート

### パターン名

ファイル: `パス`

```code
テンプレートコード
```

## 注意事項

- 注意点
```

## 新しい戦略ファイルの作成

既存カテゴリに収まらない場合、新しいファイルを作成する。

- ファイル名: `{category}.md`
- 1ファイル = 1つの開発タスクカテゴリ
- 長すぎる場合は分割（目安: 200行以上）
